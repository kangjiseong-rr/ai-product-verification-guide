# 공개 전 점검표

대외 공개용 문서 사이트로 게시하기 전에 아래 항목을 확인하세요.

## 1. 민감정보 점검

- [ ] 실제 API Key, Access Token, 비밀키가 포함되어 있지 않다.
- [ ] 내부망 URL, 사설 IP, 관리자 계정 정보가 포함되어 있지 않다.
- [ ] 개인정보가 포함된 캡처 이미지나 샘플 데이터가 없다.
- [ ] 로그 예시는 사용자 ID, 파일명, 내부 경로, 토큰을 마스킹했다.
- [ ] 비공개 고객사명, 기관명, 사업정보가 포함되어 있지 않다.

## 2. 문구 점검

- [ ] “확정”, “보장”, “무조건 통과”처럼 오해될 수 있는 표현을 제거했다.
- [ ] 제도명, 법적 근거, 담당 기관명, 문의처가 최신 정보인지 확인했다.
- [ ] 내부 심사용 표현을 대외 안내용 표현으로 바꾸었다.
- [ ] 예시는 실제 제품명을 사용하지 않거나 공개 가능한 범위로 익명화했다.

## 3. 기술 문서 점검

- [ ] 링크가 정상 동작한다.
- [ ] 표가 모바일 화면에서 읽을 수 있다.
- [ ] 이미지가 깨지지 않는다.
- [ ] 코드 블록에 실제 비밀값이 없다.
- [ ] `mkdocs build --strict`가 성공한다.

## 4. GitHub 저장소 점검

- [ ] 저장소가 public으로 공개되어도 되는 내용만 포함한다.
- [ ] `.env`, 인증서, 원본 로그 등은 `.gitignore`에 포함되어 있다.
- [ ] GitHub Pages 배포 방식이 GitHub Actions로 설정되어 있다.
- [ ] README에 로컬 미리보기 및 배포 방법이 포함되어 있다.

## 5. 최종 승인

- [ ] 담당 부서 검토 완료
- [ ] 법무 또는 정책 검토 필요 여부 확인
- [ ] 대외 공개 일정 및 담당자 확정
- [ ] 문의 채널 확정
