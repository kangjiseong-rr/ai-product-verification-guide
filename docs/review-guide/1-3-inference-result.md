# 1.3 추론 결과 확인

## 확인 질문

**입력 데이터가 모델을 거쳐 유의미한 결과, 예측·생성 등을 내는가?**

## 항목

[항목 1.3 추론 결과 확인]

## 서류 확인 사항

### STEP 1

신청서에 **입출력 데이터에 대한 정보**가 기재되어 있는지 확인합니다.

- 입력 형식이 명확히 기재되어 있는지 확인합니다.
    - 예: 이미지 `224×224×3 RGB`, JSON 텍스트 등
- 출력 형식이 명확히 기재되어 있는지 확인합니다.
    - 예: 클래스, 벡터, 생성 텍스트 등

## 현장 확인 사항

### STEP 1. 입력값 변경에 따른 출력 변화 확인

입력값 변경에 따라 출력값이 모델의 학습된 패턴에 따라 합리적으로 변화하는지 확인합니다.

!!! note
    단순 매핑 또는 하드코딩이 아닌지 검증하기 위한 확인입니다.

확인 방법은 다음과 같습니다.

#### 반복 입력을 통한 일관성 확인

동일 입력을 수회 반복하여 다음 사항을 확인합니다.

- 결정론적 모델은 동일 출력을 내는지 확인합니다.
- 확률적 모델, 예를 들어 생성형 AI 등은 합리적 분포 내 출력을 내는지 확인합니다.

#### Out-of-Sample 입력 처리

학습 데이터에 없는 신규 입력을 투입했을 때 다음 사항을 확인합니다.

- 시스템이 적절한 예외 처리, 즉 Fallback을 하는지 확인합니다.
- 예측 신뢰도, 즉 Confidence Score를 낮게 산출하는지 확인합니다.
- 신규 입력에 대해 확인 가능한 결과를 산출하는지 확인합니다.

#### 입력 변화에 따른 출력 경향성 검증

입력값의 수치나 강도가 단계별로 변할 때, 출력값도 모델이 학습한 패턴에 따라 합리적으로 변하는지 검증합니다.

### STEP 2. 모델 연산 시 리소스 사용량 증가 확인

모델 연산 시 리소스 사용량 증가를 확인합니다.

- 추론 시 GPU 또는 CPU 연산 리소스 사용량이 모델 호출 시점과 일치하는지 확인합니다.
- 연산 자원 사용 패턴을 모니터링합니다.

## 판정 기준

아래에 모두 해당되면 **PASS**로 판정합니다.

1. 입력값 변경에 따라 출력값이 합리적으로 변하는 것이 확인된 경우
2. Out-of-Sample 입력 처리가 정상적인 경우
3. 추론 시 리소스 사용량 증가가 확인된 경우

## 유의사항

인공지능 모델이 아닌 단순 매핑이나 하드코딩된 룰 기반 알고리즘이 아닌지 검증하기 위해 적절한 검증 기법을 추가하여 확인할 수 있습니다.
