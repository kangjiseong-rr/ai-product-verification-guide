# 공개 전 점검표

대외 공개 전 아래 항목을 확인합니다.

## 문서 내용 점검

- [ ] 제도명, 기준명, 항목 번호가 최신 기준과 일치하는지 확인
- [ ] 법적 근거 조항이 정확한지 확인
- [ ] 공개 대상이 신청기업용인지, 확인자용인지, 내부 심사용인지 명확히 구분
- [ ] 내부 검토용 표현, 임시 문구, 미확정 일정이 남아 있지 않은지 확인
- [ ] 예시 기업명, 제품명, 담당자명, 연락처가 공개 가능한 정보인지 확인

## 보안·개인정보 점검

- [ ] API Key, Access Token, Authorization Header가 노출되지 않았는지 확인
- [ ] 호출 로그 예시에 실제 사용자 ID, 파일 ID, 내부 서버 주소가 포함되지 않았는지 확인
- [ ] 캡처 이미지에 개인정보, 내부 시스템 URL, 계정 정보가 없는지 확인
- [ ] 신청서 예시 또는 증빙 예시에 실제 기업의 영업비밀이 포함되지 않았는지 확인

## GitHub 공개 점검

- [ ] GitHub 저장소가 public으로 공개되어도 되는 파일만 포함했는지 확인
- [ ] 원문 PDF 또는 원문 페이지 이미지 공개 여부를 별도로 결정
- [ ] GitHub Pages 주소, 저장소명, 기관명 표기가 적절한지 확인
- [ ] README에 운영 주체, 문의처, 문서 버전, 업데이트 기준을 기재했는지 확인

## 운영 점검

- [ ] 문서 담당자와 승인 절차를 정했는지 확인
- [ ] 변경 이력 관리 방식, 예를 들어 `CHANGELOG.md`를 정했는지 확인
- [ ] 문의 접수 채널을 정했는지 확인
- [ ] 정기 업데이트 주기를 정했는지 확인
