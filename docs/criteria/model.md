# 인공지능 모델 확인

인공지능 모델 확인은 제품·서비스 안에 실제 AI 모델 또는 AI 처리 연산체계가 적용되어 있는지를 확인하는 영역입니다.

## 항목 1.1 인공지능 적용

**질문:** 머신러닝, 딥러닝, LLM 등 인공지능 모델이 적용되어 있는가?

### 제출 전 확인할 사항

1. 신청서에 AI 기능 명세가 작성되어 있는지 확인합니다.
2. 기능별 AI 구현 명세를 바탕으로 사용 모델을 목록화합니다.
3. 사용 모델이 인정 인공지능 모델 목록에 포함되는지 확인합니다.

### 판정 방향

아래 중 하나 이상에 해당하면 통과 가능성이 있습니다.

- 인정 인공지능 모델을 사용한 경우
- 인정 모델 목록에 없더라도 학습 및 추론 특성이 있는 모델임을 증빙한 경우

### 모델 목록화 예시

| 기능명 | AI 구현 방식 | 모델 | 비고 |
|---|---|---|---|
| 챗봇 응답 생성 | 파인튜닝 | Qwen 계열 모델 | 한국어 특화 튜닝 |
| 판매량 예측 | 자체 학습 모델 | scikit-learn 기반 회귀 모델 | 수요 예측용 |

!!! note "유의사항"
    인정 인공지능 모델을 사용하지 않은 경우에는 항목 1.2에서 학습·추론 특성 존재 여부를 추가로 확인할 수 있습니다.

## 항목 1.2 학습·추론 특성 존재

**질문:** 모델에 학습 또는 추론 등 인공지능 특성이 존재하는가?

### 서류 확인 관점

- 항목 1.1에서 인정 인공지능 모델을 사용한 경우 자동 통과될 수 있습니다.
- 인정 목록에 없는 모델을 사용한 경우 현장 확인이 필요할 수 있습니다.

### 현장 확인 관점

다음과 같은 시연을 통해 학습·추론 특성을 확인할 수 있습니다.

- 학습 데이터 또는 모델 변경 시 출력값이 달라질 수 있음을 확인
- 모델 학습, 정확도 검증 등 AI 모델 개발 과정을 시연
- 학습되지 않은 신규 데이터에 대해 추론 결과가 생성됨을 확인
- GPU, NPU 등 연산 자원 사용량을 모니터링하여 실제 학습 또는 추론 연산 발생 여부 확인

### 판정 방향

아래 중 하나 이상에 해당하면 통과 가능성이 있습니다.

- 인정 인공지능 모델을 사용함
- 시연을 통해 학습 및 추론 특성이 확인됨

## 항목 1.3 추론 결과 확인

**질문:** 입력 데이터가 모델을 거쳐 유의미한 결과를 내는가?

### 서류 확인 관점

신청서에 입력과 출력 데이터 형식이 명확히 기재되어 있어야 합니다.

| 구분 | 예시 |
|---|---|
| 입력 형식 | 이미지 224×224×3 RGB, JSON 텍스트, 음성 파일, 센서 데이터 |
| 출력 형식 | 클래스, 벡터, 생성 텍스트, 예측값, 신뢰도 점수 |

### 현장 확인 관점

다음 내용을 확인할 수 있습니다.

- 입력값 변경에 따라 출력값이 모델의 학습 패턴에 맞게 합리적으로 변하는지 확인
- 동일 입력을 반복했을 때 결정론적 모델은 동일 출력, 생성형 AI 등 확률적 모델은 합리적 분포 내 출력을 내는지 확인
- 학습 데이터에 없는 신규 입력에 대해 예외 처리, Fallback, 낮은 신뢰도 점수 산출 등 적절한 처리가 이루어지는지 확인
- 추론 시점과 CPU/GPU/NPU 사용량 증가 시점이 일치하는지 확인

### 판정 방향

아래에 모두 해당하면 통과 가능성이 있습니다.

1. 입력값 변경에 따라 출력값이 합리적으로 변함
2. Out-of-Sample 입력 처리가 정상적으로 이루어짐
3. 추론 시 리소스 사용량 증가가 확인됨

!!! warning "단순 규칙 기반과 구분"
    단순 매핑, 하드코딩, IF 조건문 기반 룰만으로 동작하는 경우에는 AI 모델 적용으로 보기 어려울 수 있습니다.
