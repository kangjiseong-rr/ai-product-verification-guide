# 인공지능 기능 확인

인공지능 기능 확인은 AI 모델이 제품 안에서 핵심 기능과 의미 있게 결합되어 동작하는지를 확인하는 영역입니다.

## 항목 2.1 구조

**질문:** 아키텍처 설계도상 AI 모델의 위치가 명확하며, 제품 내부 로직과 필수불가결하게 결합되어 동작하는가?

### 서류 확인 관점

아키텍처 설계도에는 다음 내용이 표현되어야 합니다.

- AI 모델 또는 AI 엔진의 위치
- AI 모델이 구동되는 운영 환경: 서버, Cloud, 외부 AI API, 사용자 단말 등
- AI 모델과 제품 핵심 기능 간 연결 구조
- 외부 API를 사용하는 경우 외부망 경계와 연동 방식

### 판정 방향

아래에 모두 해당하면 통과 가능성이 있습니다.

1. AI 모델 위치가 명확히 표시됨
2. AI 모델 컴포넌트를 삭제할 경우 핵심 기능 수행이 불가능하거나 성능이 저하됨

!!! tip "설계도 작성 팁"
    단순히 “AI API 사용”이라고 쓰기보다, 사용자 입력 → 전처리 → AI 모델 호출 → 결과 처리 → 화면/업무 로직 반영의 흐름을 보여주는 것이 좋습니다.

## 항목 2.2 생산성 향상

**질문:** AI 모델이 해당 제품의 성능 향상이나 편의성에 기여하는가?

### AI 적용 목적 유형

| 유형 | 설명 |
|---|---|
| 기능 추가 | AI 적용을 통해 기존 제품에 없던 핵심 기능이 추가된 경우 |
| 기능 개선 | AI 적용을 통해 출력의 정확도 및 품질을 개선한 경우 |
| 사용자 편의성 | 핵심 기능에서 사용자 입력·조작 단계를 줄이고 반복작업을 감소시킨 경우 |
| 접근성 개선 | 텍스트, 음성, 이미지 등 다양한 입력 방식을 지원하는 경우 |
| 자원 효율화 | CPU/GPU 사용량, 응답 시간, 처리량 등을 개선한 경우 |

### 현장 확인 관점

- 기능 추가 또는 기능 개선: AI가 핵심 기능에 어떤 변화를 만들었는지 확인
- 사용자 편의성, 접근성 개선, 자원 효율화: 개선 결과에 대한 증빙 확인
- 예: 작업 단계 감소 비교, 응답 시간 비교, GPU 사용량 비교, 처리량 비교 등

### 판정 방향

아래에 모두 해당하면 통과 가능성이 있습니다.

1. AI 적용 목적이 기능 추가, 기능 개선, 사용자 편의성, 접근성 개선, 자원 효율화 중 하나 이상에 해당됨
2. 현장 확인 또는 증빙자료를 통해 개선 내용이 확인됨

## 항목 2.3 핵심 AI 기능 확인

**질문:** AI 모델이 적용된 핵심 기능이 정상적으로 동작하는가?

### 현장 확인 관점

- 정상 입력 데이터의 영역, 구간, 조건 내에서 AI 기능이 정상 수행되는지 확인
- 비정상 입력 데이터에 대해서는 예외 처리되는지 확인
- 학습 기능이 있는 경우 학습 데이터 추가 또는 업데이트 후 기존 기능이 유지되고 신규 데이터가 반영되는지 확인

### 판정 방향

아래에 모두 해당하면 통과 가능성이 있습니다.

1. AI 기능이 정상적으로 동작함
2. 학습 기능이 있는 경우 학습으로 인해 출력값이 변경됨이 확인됨

!!! note "테스트 기법"
    기능 확인을 위해 경계값 분석, 동등 클래스 분할 등 일반적인 소프트웨어 테스트 기법을 적용할 수 있습니다.
