# 제출 전 준비사항

신청 전에는 “AI가 적용되었다”는 설명만으로는 부족합니다. 제품 안에서 AI가 어떤 역할을 하고, 어떤 입력을 받아 어떤 결과를 내며, 핵심 기능과 어떻게 결합되어 있는지 확인할 수 있어야 합니다.

## 1. AI 기능 명세 준비

신청서에는 AI가 적용된 기능을 기능 단위로 구분하여 작성하는 것이 좋습니다.

| 항목 | 작성 예시 |
|---|---|
| AI 기능명 | 문서 자동 요약, 이상 탐지, 객체 인식, 수요 예측 |
| AI 역할 | 분류, 예측, 생성, 추천, 탐지, 추출 |
| 사용자 입력 | 이미지, 텍스트, 음성, 센서 데이터, 로그 데이터 |
| 시스템 출력 | 클래스, 점수, 생성 텍스트, 알림, 예측값 |
| AI 핵심기능 여부 | 제품 핵심 기능인지, 보조 기능인지 구분 |

## 2. AI 구현 방식 정리

다음 중 어떤 방식으로 구현되었는지 명확히 정리합니다.

- 자체 학습 모델
- 오픈소스 모델 활용
- 파인튜닝 모델
- 상용 AI API 연동
- 온디바이스 AI 또는 로컬 추론
- 클라우드 기반 AI 추론

## 3. 모델 목록화

사용한 모델은 기능별로 목록화하는 것이 좋습니다.

| 기능명 | AI 구현 방식 | 모델 | 비고 |
|---|---|---|---|
| 챗봇 응답 생성 | 파인튜닝 | Qwen 계열 모델 | 한국어 특화 튜닝 |
| 판매량 예측 | 자체 학습 모델 | scikit-learn 기반 회귀 모델 | 수요 예측용 |

!!! tip "모델명이 목록에 없는 경우"
    인정 인공지능 모델 목록에 없는 모델도 학습·추론 특성을 현장에서 확인할 수 있다면 검토될 수 있습니다.

## 4. 입출력 데이터 설명

입력과 출력 형식은 구체적으로 작성합니다.

| 구분 | 예시 |
|---|---|
| 이미지 입력 | 224×224×3 RGB 이미지 |
| 텍스트 입력 | JSON 형식의 사용자 질의 |
| 센서 입력 | 온도, 진동, 전류 등 시계열 데이터 |
| 출력 | 분류 결과, 예측값, 생성 문장, 신뢰도 점수 |

## 5. 아키텍처 설계도 준비

아키텍처에는 다음이 보여야 합니다.

- AI 모델 또는 AI 엔진의 위치
- AI 모델이 구동되는 환경: 서버, 클라우드, 외부 AI API, 사용자 단말 등
- AI 모델과 제품 핵심 기능 간 연결 관계
- 외부 AI API를 사용하는 경우 외부망 연동 경계
- AI 모델이 제거될 경우 어떤 기능이 영향을 받는지

## 6. 현장 시연 준비

현장 확인이 필요한 경우 다음 시나리오를 준비하는 것이 좋습니다.

| 시연 항목 | 준비 내용 |
|---|---|
| 정상 입력 처리 | 정상 범위 데이터에서 AI 기능이 정상 동작하는지 확인 |
| 비정상 입력 처리 | 범위를 벗어난 입력에 대해 예외 처리되는지 확인 |
| 반복 입력 | 동일 입력 반복 시 결과가 일관적인지 확인 |
| 신규 입력 | 학습 데이터에 없는 입력에 대해 적절히 처리하는지 확인 |
| 경향성 확인 | 입력 강도나 수치 변화에 따라 출력이 합리적으로 변하는지 확인 |
| 리소스 사용량 | 추론 또는 학습 시 CPU/GPU/NPU 사용량 증가 확인 |
| 외부 API 로그 | 외부 AI API 호출 로그가 생성되는지 확인 |

## 7. 공개 저장소에 넣으면 안 되는 정보

- 실제 API Key 원문
- 인증 토큰, 비밀키, 인증서
- 내부망 주소와 계정 정보
- 개인정보가 포함된 원본 데이터
- 고객사·기관 비공개 자료
- 영업비밀 또는 비공개 모델 가중치

공개 문서에는 마스킹된 예시 화면과 익명화된 샘플 데이터를 사용하세요.
