# 자주 묻는 질문

## AI API를 호출하면 모두 AI 제품으로 확인되나요?

아닙니다. 외부 AI API를 호출하더라도 제품의 핵심 기능과 기술적으로 결합되어 있어야 합니다. 단순히 범용 AI 프롬프트 창을 노출하거나 외부 LLM 응답을 화면에 그대로 출력하는 수준이라면 확인이 어려울 수 있습니다.

## 인정 인공지능 모델 목록에 없는 모델은 인정되지 않나요?

목록에 없는 모델도 학습·추론 특성이 확인되면 검토될 수 있습니다. 이 경우 모델의 학습 과정, 추론 결과, 입력 변화에 따른 출력 변화, 신규 입력 처리 등을 설명할 수 있어야 합니다.

## 규칙 기반 기능도 AI로 볼 수 있나요?

단순 IF 조건문, 평균·합계·비율 계산, 임계치 초과 알림, 키워드 매칭만으로 동작하는 기능은 AI 모델 적용으로 보기 어려울 수 있습니다.

## 현장 확인에서는 무엇을 준비해야 하나요?

정상 입력, 비정상 입력, 신규 입력, 반복 입력, 입력 변화에 따른 출력 경향성, 리소스 사용량, 외부 API 호출 로그 등을 확인할 수 있는 시연 자료를 준비하는 것이 좋습니다.

## 외부 모델 API Key를 증빙으로 제출해야 하나요?

API Key 존재 여부는 확인 대상이 될 수 있지만, 공개 문서나 공개 저장소에는 실제 Key 원문을 포함하면 안 됩니다. 화면 예시나 로그는 반드시 마스킹해야 합니다.

## AI 챗봇이 있으면 확인 대상인가요?

챗봇이 제품 사용법 안내, FAQ, 메뉴 안내 등 보조 기능에 그치고 제품의 본래 기능과 필수적으로 결합되어 있지 않다면 확인이 어려울 수 있습니다. 반대로 RAG, 문서 자동 검토, 업무 처리 자동화 등 핵심 기능과 결합되어 있다면 확인 가능성이 높아질 수 있습니다.

## 온디바이스 AI나 로컬 추론도 대상인가요?

가능합니다. 이 경우 AI 모델이 기기 또는 로컬 환경에서 실제로 추론을 수행하고, 제품 핵심 기능과 결합되어 있음을 구조도와 현장 시연으로 설명할 수 있어야 합니다.
