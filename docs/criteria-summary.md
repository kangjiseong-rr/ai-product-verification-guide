# 확인 기준 한눈에 보기

아래 기준을 만족하는 경우 AI 제품·서비스로 확인될 수 있습니다.

| 구분 | 번호 | 확인 내용 | 확인 방식 |
|---|---:|---|---|
| 인공지능 모델 확인 | 1.1 | 제품·서비스에 인공지능처리 연산체계가 적용되어 있는가? | 서류 |
| 인공지능 모델 확인 | 1.2 | 적용된 인공지능처리 연산체계가 학습·추론 등 AI로서의 특성을 가지고 있는가? | 서류, 현장 |
| 인공지능 모델 확인 | 1.3 | 입력 데이터가 AI 처리 연산체계를 거쳐 유의미한 결과를 출력하는가? | 서류, 현장 |
| 인공지능 기능 확인 | 2.1 | 구조도상 AI 또는 AI 기술의 위치가 명확하며 제품·서비스의 구동 체계와 필수불가결하게 결합되어 동작하는가? | 서류 |
| 인공지능 기능 확인 | 2.2 | AI 처리 연산체계가 제품·서비스의 기능, 성능, 편의성 등에 기여하는가? | 서류, 현장 |
| 인공지능 기능 확인 | 2.3 | AI 처리 연산체계가 적용된 주요 기능이 정상적으로 동작하는가? | 현장 |
| 외부 모델 연동 확인 | 3.1 | 외부 모델과 연동된 경우 호출 로그나 API Key 등을 확인할 수 있는가? | 서류, 현장 |

## 한 문장으로 이해하기

- **1번 영역**은 “AI 모델이 실제로 있는가?”를 봅니다.
- **2번 영역**은 “그 AI가 제품의 핵심 기능에 의미 있게 붙어 있는가?”를 봅니다.
- **3번 영역**은 “외부 AI 모델을 쓴다면 실제 연동 흔적을 확인할 수 있는가?”를 봅니다.

## 신청자가 특히 준비해야 할 것

| 준비 항목 | 설명 |
|---|---|
| AI 기능 명세 | 어떤 기능에 AI가 쓰이는지 기능 단위로 설명 |
| AI 구현 방식 | 자체 학습, 파인튜닝, 오픈소스 모델, 외부 API 연동 등 구현 방식 기재 |
| 모델 목록 | 사용 모델명, 라이브러리, 모듈명, 버전, 역할 기재 |
| 입출력 데이터 | 입력 형식과 출력 형식 명시 |
| 아키텍처 설계도 | AI 모델 위치, 운영 환경, 외부 API 연동 구조 표시 |
| 시연 시나리오 | 정상 입력, 예외 입력, 신규 입력, 리소스 사용량 확인 등 준비 |
| 증빙자료 | 로그, 성능 비교표, 사용자 매뉴얼, 제품 설명서 등 |
