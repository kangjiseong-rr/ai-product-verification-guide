# AI 적용 확인 불가 사례

다음은 AI 적용 확인이 어려울 수 있는 사례입니다. 핵심 판단 기준은 AI가 제품의 본래 기능과 필수적으로 결합되어 있는지, 그리고 AI 모델의 학습·추론 결과가 제품 기능에 실질적으로 기여하는지입니다.

## 1. 범용 하드웨어에 외부 AI 앱을 단순 설치한 경우

- 일반 PC, 키오스크 등을 납품하면서 AI 바로가기 또는 앱만 설치한 경우
- AI 기능이 하드웨어 제어 또는 핵심 기능과 연결되지 않은 경우
- AI 앱이나 API를 제거해도 제품의 주요 기능이 그대로 동작하는 경우

## 2. 도움말·FAQ·Q&A 단순 연동 제품

- 제품 사용법을 알려주는 AI 챗봇 기능만 제공하는 경우
- 메뉴 위치를 안내하는 AI 기능만 제공하는 경우
- FAQ 검색에만 LLM을 연동한 경우
- ERP 또는 업무 시스템에 고객 응대용 챗봇을 부가 기능으로 탑재한 경우
- AI를 제거해도 제품 본래 기능 수행에 지장이 없는 경우

## 3. 외부 AI API 단순 호출 제품

- 외부 LLM 응답 결과를 화면에 단순 텍스트 출력만 하는 경우
- 확인 대상 제품의 처리 로직 없이 범용 AI 프롬프트 창을 그대로 노출한 경우
- 제품 핵심 기능과 기술적 결합성이 약한 경우

## 4. 학습·추론 없이 규칙 기반으로 동작하는 제품

- IF 조건문 기반 추천 기능을 “AI 추천”으로 표시한 경우
- 평균, 합계, 비율, 임계치 초과 알림을 “AI 분석”으로 표시한 경우
- 사전 정의된 시나리오 챗봇을 “AI 챗봇”으로 표시한 경우
- 데이터 학습이나 추론 없이 단순 키워드 매칭을 “AI 검색”으로 표시한 경우

## 확인이 어려운 표현을 피하는 방법

| 피해야 할 표현 | 보완 방향 |
|---|---|
| AI 기능 탑재 | 어떤 모델이 어떤 입력을 받아 어떤 결과를 내는지 설명 |
| AI API 사용 | 외부 API 결과가 제품 핵심 기능에 어떻게 반영되는지 설명 |
| AI 분석 | 학습·추론 과정, 입력 데이터, 출력 결과, 검증 방법 설명 |
| AI 추천 | 규칙 기반 추천인지, 모델 기반 추천인지 구분 |
